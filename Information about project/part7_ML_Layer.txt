# ABOUT_PROJECT.TXT

# PART 7 — MACHINE LEARNING CORE, ENSEMBLE ARCHITECTURE, META-LABELING, FEATURE SELECTION AND ALPHA GENERATION

---

# 202. MACHINE LEARNING LAYER OVERVIEW

The Machine Learning Layer is the central prediction engine of the project.

All previous layers exist primarily to provide structured information to this layer.

The ML Layer does not collect data.

The ML Layer does not execute trades.

The ML Layer does not manage risk.

Its purpose is simple:

Transform intelligence into probabilistic forecasts.

---

# 203. CORE PHILOSOPHY

The project deliberately avoids relying on a single model.

Reason:

Markets are non-stationary.

A model that performs well during:

Bull Markets

may completely fail during:

Bear Markets

Sideways Markets

High Volatility Markets

Therefore:

Model diversification is mandatory.

---

# 204. WHY ENSEMBLES ARE USED

Single models suffer from:

Overfitting

Regime Bias

Instability

Model Drift

Ensembles reduce these weaknesses.

The objective is not:

Maximum Accuracy

The objective is:

Stable Alpha Generation.

---

# 205. PRIMARY ML OBJECTIVE

Predict future market behavior using information from:

Price

Volume

OI

FII/DII

News

Macro

Mathematical Layer

Microstructure Layer

The output becomes:

Probabilistic market forecasts.

---

# 206. PREDICTION TARGETS

The system may support:

Direction Prediction

Return Prediction

Volatility Prediction

Trade Success Probability

Regime Prediction

Future versions may support all simultaneously.

---

# 207. PRIMARY MODEL FAMILY

The project will use a diversified ensemble.

Primary Models:

XGBoost

LightGBM

LSTM

GRU

Future expansion remains possible.

---

# 208. WHY XGBOOST

XGBoost remains one of the strongest models in quantitative finance.

Advantages:

Fast

Robust

Handles Nonlinear Relationships

Handles Missing Values

Provides Feature Importance

Works well with tabular features

---

# 209. WHY LIGHTGBM

LightGBM offers:

Faster Training

Lower Memory Usage

Excellent Tabular Performance

High Scalability

Often performs similarly to XGBoost.

---

# 210. WHY SEQUENCE MODELS EXIST

Markets possess memory.

Yesterday matters.

Last week matters.

Last month matters.

Tree models struggle with sequential memory.

Sequence models address this weakness.

---

# 211. LSTM

Long Short-Term Memory Networks.

Purpose:

Capture temporal dependencies.

Useful when:

Historical sequence structure matters.

---

# 212. GRU

Gated Recurrent Unit.

Purpose:

Simpler alternative to LSTM.

Benefits:

Faster

Lower Parameter Count

Often similar performance.

---

# 213. MODEL DIVERSIFICATION PHILOSOPHY

Each model specializes in different behavior.

Example:

XGBoost:

Excellent at nonlinear tabular relationships.

LSTM:

Excellent at temporal memory.

GRU:

Excellent balance between speed and memory.

Combining them improves robustness.

---

# 214. ENSEMBLE OUTPUT

Individual models generate forecasts.

The ensemble combines them into:

Unified Prediction Probabilities.

The ensemble is generally more stable than any individual model.

---

# 215. PRIMARY MODEL RESPONSIBILITY

The primary ensemble performs only one task:

Directional Forecasting.

Example:

Class 2:

Strong Bullish

Class 1:

Neutral

Class 0:

Strong Bearish

Nothing else.

---

# 216. WHY THE PRIMARY MODEL IS RESTRICTED

Many trading systems fail because a single model attempts to solve:

Direction

Risk

Position Sizing

Execution

simultaneously.

This project separates responsibilities.

for

T predictions.

# 221. TRIPLE-BARRIER LABELING

The project intends to use:

Triple Barrier Labeling.

Instead of:

Arbitrary Up/Down Labels.

---

# 222. WHY TRIPLE-BARRIER LABELING

Traditional labels are noisy.

Triple Barrier introduces:

Take Profit Barrier

Stop Loss Barrier

Time Expiration Barrier

This produces cleaner labels.

---

# 223. BENEFITS OF TRIPLE BARRIER

Improves:

Signal Quality

Risk Awareness

Label Stability

Strategy Realism

---

# 224. META-LABELING OVERVIEW

Meta-labeling is one of the most important concepts in the project.

Purpose:

Evaluate prediction quality.

Not direction.

Quality.

---

# 225. PRIMARY VS META MODEL

Primary Model:

Predicts Direction.

Meta Model:

Predicts Probability Of Success.

These are completely different tasks.

---

# 226. EXAMPLE

Primary Model:

BUY

Meta Model:

34% Success Probability

Result:

Trade Rejected.

---

Primary Model:

BUY

Meta Model:

89% Success Probability

Result:

Trade Accepted.

---

# 227. WHY META-LABELING EXISTS

Most false signals originate from:

Low-confidence environments.

Meta-labeling attempts to identify:

Bad trades

before execution.

---

# 228. META MODEL INPUTS

Potential Inputs:

These are contextual features.

---

# 229. IMPORTANT RULE

Primary Model Features

---

# 230. CALIBRATION THRESHOLD

Meta-labeler outputs:

0 → 1

representing confidence.

Example:

0.35

0.61

0.82

0.94

A threshold determines whether a trade survives.

# 244. DEFLATED SHARPE RATIO

One of the most important validation metrics.

Purpose:

Detect false discoveries.

Many strategies appear profitable due to randomness.

DSR adjusts for:

Multiple Testing Bias.

---

# 245. WHY DSR IS REQUIRED

The project will test:

Hundreds

or

Thousands

of configurations.

Without DSR:

False Alpha becomes likely.
