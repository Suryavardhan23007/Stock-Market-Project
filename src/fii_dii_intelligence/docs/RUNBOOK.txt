# FII/DII Intelligence Layer - Operational Runbook

## 1. Which file collects data?
The core ingestion script is `src/fii_dii_intelligence/ingestion/collector.py`. 
It validates arithmetic, stores raw archives, triggers the analytics engine, and updates the final intelligence table.

## 2. How to start collection?
Currently, this is a daily cron-triggered batch job, not a continuous daemon like the OI layer.
It should be triggered at 19:30 IST daily (after NSE publishes final provisional flows).
To manually ingest a day's flow (for instance, via a python REPL):

```python
from datetime import datetime
from src.database.connection import SessionLocal
from src.fii_dii_intelligence.ingestion.collector import FIIDIICollector

db = SessionLocal()
collector = FIIDIICollector(db)
collector.ingest_daily_flow(
    trade_date=datetime(2026, 6, 17),
    fii_buy=10000, fii_sell=8000, fii_net=2000,
    dii_buy=5000, dii_sell=6000, dii_net=-1000,
    source="NSE"
)
```

## 3. How to stop collection?
Disable the system cron job or PM2 scheduled task governing the 19:30 execution.

## 4. How to verify collection?
Connect to the PostgreSQL database and execute:
```sql
SELECT trade_date, fii_net, fii_5d_flow, fii_buy_streak 
FROM institutional_flow_intelligence 
ORDER BY trade_date DESC LIMIT 5;
```

## 5. How to verify missing days?
Cross-reference the `trade_date` count in `institutional_flow_intelligence` against the number of official NSE trading days in the current month. If the script fails, you will see a gap in the dates. Upsert logic guarantees you can simply re-run the collector script for the missing date.
